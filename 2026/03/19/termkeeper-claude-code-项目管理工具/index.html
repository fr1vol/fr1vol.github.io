TermKeeper：解决 Claude Code 项目移动后的失忆问题

John Doe
2026-03-19
工具
Claude-Code, Python

问题背景

使用 Claude Code 开发时，你是否遇到过这样的困扰：

当你把一个项目目录移动到新位置（比如从 /home/user/old/project 移动到 /home/user/new/project），重新打开 Claude 后，它完全不认识这个项目了——所有的对话历史、上下文记忆都找不到了。

这是因为 Claude Code 用路径作为项目标识符：

虽然内容没变，但路径变了，Claude 就认为这是一个全新的项目。

解决方案：TermKeeper

TermKeeper (tk) 是一个专门解决这个问题的工具，通过重命名 Claude 的项目目录来恢复记忆。

核心功能

命令 功能
tk . 或 tk -m 自动迁移（智能检测最佳匹配）
tk -m <path> 显式迁移（直接指定旧路径）
tk migrate 交互式迁移（列出所有项目）
tk -a 归档会话（JSONL 转 Markdown）

智能匹配算法

tk . 使用多维度评分自动找到旧项目：

匹配条件 分数
目录名完全匹配 +100
目录名部分匹配 +50
归档包含当前路径 +80
归档包含当前 slug +60
归档包含目录名 +30

增量归档

基于 SHA256 hash 检测文件变更，避免重复处理：

  1. 计算源 JSONL 文件的 SHA256
  2. 读取现有归档的 source_hash
  3. hash 相同 → 跳过
  4. hash 不同 → 更新归档

项目特点

  • 零依赖：仅使用 Python 标准库
  • 极简：单文件，约 600 行代码
  • 可靠：基于 hash 检测，比 mtime 更准确
  • 开源：MIT License
